# Code Quality Review Instruction

## Steps

1. Read the implementation report from {report_dir}/implementation.md (if available)
2. Read the plan from {report_dir}/plan.md for context
3. Review the actual code changes, focusing exclusively on:
   - Code quality, readability, and maintainability
   - Test coverage and correctness
   - Error handling and edge cases
   - API design and naming conventions
   - Code duplication and abstraction quality
   - Adherence to project coding standards

## Routing

Output one of the following:
- `approved` — Code quality meets standards
- `needs_fix` — Changes needed (provide specific, actionable feedback)
